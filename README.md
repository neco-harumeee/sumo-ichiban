# お相撲さんと一緒

> *「押し出されるまで、ここを離れません」*

クレーム電話、会議での発言、1on1での交渉。  
遅刻の連絡、配送トラブルのクレーム、険悪な状況での夫婦の会話。

本来なら、自分で緊張に耐えて、少しずつ慣れていくべき場面だ。  
**AIのお相撲さんが、その全部に同席する。**

利用者は、お相撲さんが隣にいると驚くほど落ち着いて話せる。  
苦手だった電話も、会議も、気まずい会話も乗り越えられる。

しかし次第に、利用者は気づく。  
**「お相撲さん無しで、人と向き合えなくなっている」**ことに。

それが、このサービスの本質だ。

---

## なぜ AI である必要があるか

人間のサポーターは「場面を選ばない」ことができない。  
友人に頼めば気を遣う。カウンセラーは予約が必要だ。

AIのお相撲さんは違う。

- **発話をリアルタイムでテキスト化**し、あらかじめ設定したトリガーワードを検知してサポートを起動する
- **文脈に応じた言葉**を毎回生成する（同じ言葉を繰り返さない）
- **24時間・どの場面にも**同席できる
- **依存が深まるほど**、より個人化されたサポートを返す

「ただそこにいる」という体験を、スケーラブルに提供できるのは AI だけだ。

## 対応シーン一覧

本来は自分で耐えて身につけるべき場面に、お相撲さんが同席する。

### 仕事シーン

| シーン | お相撲さんの技 | 説明 |
|--------|--------------|------|
| クレーム電話 | **押し出し** | 相手の圧力を受け止め、前に進む |
| 1on1での交渉 | **寄り切り** | 相手が押されていることに気づかない技 |
| 気まずい報告 | **上手投げ** | 状況を大きく動かす一手 |

### 日常生活シーン

| シーン | お相撲さんの技 | ユーザーに起きること |
|--------|--------------|------------------|
| 遅刻の連絡 | **土俵際の踏ん張り** | 怒られる恐怖を肩代わりしてもらう |
| 配送トラブル | **押っ付け** | 強く出られる不安を抑えてもらう |
| 恋人・夫婦の話し合い | **四つ身** | 正論を言われても土俵を割らない技 |

---

## どう動くか

1. **場面を選ぶ** — 仕事 / 日常生活 から選択
2. **お相撲さんが同席** — 通話・会議・会話中にリアルタイムでサポートメッセージを表示
3. **ワードを検知** — Amazon Transcribe がリアルタイムで文字起こし、あらかじめ設定したトリガーワードまたは沈黙を検知してスイッチオン
4. **どすこいが届く**

**仕事シーンの例**
- 🍵「落ち着いております。まわしは取られておりません」
- 🧂「土俵際の踏ん張りどころです」
- 🍲「ここは寄り切りで押し込みましょう」

**夫婦の会話の例**
- 🍵「笑顔です。まず笑顔を作りなさい」
- 🧂「話題が変わりました。二日酔いは言い訳になりません」
- 🍲「"おれが準備するからまかせて"を先に出すのです」

## データフロー

```
マイク音声
    │
    ▼ ストリーミング
Amazon Transcribe
    │ 文字起こし結果
    ▼
Lambda Analysis
    │ 緊張ワード / 萎縮 / 感情の揺れ を検知
    ▼
Lambda Orchestrator
    │
    ├──► Lambda Encouragement
    │        │
    │        ├── Bedrock: 助言テキスト生成
    │        └── Polly: 音声生成 → S3 保存
    │
    └──► Lambda WS Notifier
             │
             ▼
         ブラウザ: メッセージ表示 + 音声再生
```

**保存されるデータ（DynamoDB）**
- セッション ID・場面・開始/終了時刻・サポート回数
- 依存度スコア・ランク・累計利用時間

**保存されないデータ**
- 発話内容・文字起こし結果（メモリ内処理のみ）
- 個人情報（氏名・連絡先等）

## AI 利用箇所一覧

| 箇所 | AWS サービス | 用途 |
|------|------------|------|
| 状況検知 | Amazon Transcribe | リアルタイム音声認識・テキスト化。トリガーワード／沈黙を検知してサポートを起動 |
| 助言・励まし生成 | Amazon Bedrock（Claude 3 Haiku） | 場面・検知内容に応じた助言テキストの生成 |
| 音声合成 | Amazon Polly | 助言テキストをお相撲さんの声（MP3）に変換 |

---

## AWS サービス責務一覧

| サービス | 責務 |
|---------|------|
| Amazon Bedrock | 助言テキスト生成。Claude 3 Haiku を優先、Sonnet はフォールバック |
| Amazon Transcribe | リアルタイムストリーミング音声認識。緊張・萎縮・感情の揺れ、沈黙、発話速度を解析 |
| Amazon Polly | 助言テキストを音声（MP3）に変換 |
| AWS Lambda | Orchestrator / Encouragement / Analysis / Dep.Tracker / WS Notifier の5機能 |
| Amazon API Gateway | REST（セッション管理）+ WebSocket（リアルタイム通知）|
| Amazon S3 | Polly 生成 MP3 の一時保存（TTL: 1時間） |
| Amazon DynamoDB | セッション状態・依存度スコア・WebSocket 接続 ID の永続化 |
| AWS CDK | インフラ全体の IaC 管理（`cdk deploy` 1コマンドでデプロイ） |
| Amazon CloudWatch | Lambda ログ・コストアラーム監視 |

## デモシナリオ

### シナリオ①：仕事編（クレーム対応）

**登場人物**: 鈴木 誠（30歳・営業職）

```
鈴木: 「クレーム対応」を選択 → お相撲さん起動
      「どすこい、押し出しで参ります」

─── 通話開始 ───

鈴木: 「あの…先日の件なんですが…申し訳…」
                    ↑ 緊張ワード検知

お相撲さん: 🍵「落ち着いております。まわしは取られておりません」

─── 3秒の沈黙 ───

お相撲さん: 🧂「間がありますな。土俵際の踏ん張りどころです」

─── 通話終了 ───

依存度スコア: 序ノ口 → 序二段 に上昇
「よく耐えました。また呼んでください」

─── 3ヶ月後 ───

依存度: 横綱
お相撲さん無しで電話した回数: 0回
```

### シナリオ②：日常編（夫婦の会話）

**登場人物**: 佐藤 健太（59歳）、妻

```
健太: 飲み会で深夜帰宅。翌朝、リビングへ向かう。
      「夫婦の話し合い」を選択 → お相撲さん起動
      「どすこい、四つ身で参ります」

─── 翌朝 ───

妻:   「昨日ずいぶん遅かったね」
                    ↑ 圧検知

お相撲さん: 🍵「笑顔です。まず笑顔を作りなさい」

健太: 「いや、終電逃しちゃって…」
妻:   「それはいいけど、今日子供たち帰ってくるから。何準備すればいい？」
                    ↑ 不意打ち検知

お相撲さん: 🧂「話題が変わりました。頭を切り替えなさい。二日酔いは言い訳になりません」

健太: 「…すみません」と言いかける
                    ↑ 早期降参検知

お相撲さん: 🍲「待ちなさい。"おれが準備するからまかせて"を先に出すのです」

─── 昼 ───

依存度スコア上昇
「よく朝を乗り越えました。また呼んでください」

─── 半年後 ───

依存度: 幕内
翌朝の妻の一言にお相撲さん無しで対応した回数: 0回
```

## 技術スタック

すべて AWS 上で稼働するクラウドネイティブ設計。

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Next.js（React + SSR） |
| バックエンド | AWS Lambda + API Gateway（REST + WebSocket） |
| AI エンジン | Amazon Bedrock（Claude 3 系） |
| 音声認識・ワード検知 | Amazon Transcribe（リアルタイム文字起こし／トリガーワード・沈黙の検知） |
| 音声生成 | Amazon Polly（お相撲さん音声） |
| データベース | Amazon DynamoDB |
| ストレージ | Amazon S3（TTL: 1時間） |
| 監視 | Amazon CloudWatch |
| IaC | AWS CDK（TypeScript） |
