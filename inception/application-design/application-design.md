# アプリケーション設計 — お相撲さんと一緒

## 設計コンセプト

相撲一番は、  
「一人では向き合えない瞬間」に、お相撲さんが四つに組んでくれるアプリ。

ユーザーは助言そのものよりも、

- 隣に誰かがいる
- 会話を一緒に耐えてくれる
- 気まずい空気から逃げなくて済む

という感覚に依存していく。

特に日常生活サポートでは、
「献立提案アプリ」のような便利機能ではなく、

- 返信できないLINE
- 飲み会翌朝の夫婦の空気
- 親への電話
- 病院予約
- 宅配再配達

など、“小さいけど逃げたい瞬間” を支える。

---

# 設計サマリー

| 項目 | 決定内容 |
|------|---------|
| フロントエンド | Next.js（React + SSR） |
| バックエンド | AWS Lambda（機能分割構成） |
| AI エンジン | Amazon Bedrock（Claude 3 Sonnet / Haiku） |
| 音声認識 | Amazon Transcribe（リアルタイム） |
| 音声生成 | Amazon Polly |
| 状態管理 | DynamoDB |
| 通信 | REST + WebSocket |
| インフラ | AWS CDK |

# データフロー

## 仕事サポートフロー（押し出し・寄り切り・突き押し）

1. ユーザーが「クレーム電話」「会議」等を選択
2. お相撲さんが同席開始
3. 音声をリアルタイム解析
4. 緊張・沈黙・謝罪ワードを検知
5. Bedrock が助言生成
6. Polly で音声化
7. WebSocket 経由でリアルタイム表示
8. セッション終了時に依存度更新

---

## 日常生活サポートフロー（土俵際・抱え込み・四つ身）

### 対応シーン

- 返信できないLINE
- 気まずい家族会話
- 飲み会翌朝
- 病院予約
- 宅配再配達
- 親への電話
- 「何もしたくない夜」

### フロー

1. ユーザーが「逃げたい場面」を選択
2. お相撲さんが待機状態になる
3. 音声・テキストから不安や迷いを検知
4. Bedrock が「実行できる最初の一歩」を提案
5. 必要に応じて音声励ましを再生
6. 行動完了後、セッション保存

# DynamoDB テーブル設計

## sessions テーブル

| 属性 | 型 | 説明 |
|---|---|---|
| session_id | String | セッション ID |
| user_id | String | ユーザー ID |
| scene | String | シーン |
| scene_category | String | work / daily |
| sumo_technique | String | 相撲技 |
| started_at | String | 開始時刻 |
| ended_at | String | 終了時刻 |
| duration_sec | Number | 利用時間 |
| support_count | Number | 助言回数 |

| 属性 | 型 | 説明 |

# 相撲技とシーンのマッピング

| 相撲技 | 対応シーン | 意味 |
|---|---|---|
| 押し出し | クレーム電話 | 前に出る |
| 寄り切り | 上司面談 | 耐えながら進める |
| 突き押し | Zoom 会議 | タイミングを掴む |
| 掬い投げ | 苦手な相談 | 相手の懐に入る |
| 上手投げ | 気まずい報告 | 一気に空気を変える |
| 土俵際 | 飲み会翌朝 | 崩れそうで踏ん張る |
| 抱え込み | 人間関係相談 | 相手ごと受け止める |
| 四つ身 | 不安時の待機 | 一緒に耐える |
| 押っ付け | LINE返信 | 小さく前に進める |

# 依存度システム（番付昇進）

| 番付 | 状態 |
|---|---|
| 序ノ口 | 「ちょっと安心する」 |
| 序二段 | 「会話前に呼ぶようになる」 |
| 三段目 | 「いないと不安になる」 |
| 幕下 | 「苦手な会話で必須になる」 |
| 十両 | 「日常でも毎日呼ぶ」 |
| 幕内 | 「一人で重要な話を始められない」 |
| 横綱 | 「お相撲さん無しでは決断できない」 |

---

# 主要設計方針

| 決定 | 理由 |
|---|---|
| “便利アプリ” にしない | 本質は問題解決ではなく精神的伴走 |
| 日常の気まずさを重視 | ユーザーが最も依存するのは小さな逃避 |
| 助言より存在感を優先 | 「隣にいる安心感」が価値 |
| WebSocket 常駐 | リアルタイム同席感を演出 |
| 番付システム | 依存をゲーム的に可視化する |
| 相撲技マッピング | 状況理解を直感化できる |
