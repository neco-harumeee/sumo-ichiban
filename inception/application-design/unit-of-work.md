# Unit of Work — 相撲一番（SUMO ICHIBAN）

## コンセプト

相撲一番は「問題解決アプリ」ではない。

ユーザーが本当に求めているのは、
最適解や効率ではなく、

- 気まずい空気に一緒に耐えてくれる
- 一人で向き合わなくて済む
- 「まだ土俵に残っている」と感じられる

という精神的な同席感。

そのため各ユニットも、
「便利機能」ではなく
“伴走体験” を中心に設計する。

# ユニット詳細

# U-01: frontend（仕切り）

## 説明

ユーザーがお相撲さんを呼び出し、
「四つに組む」ための入口。

ただの UI ではなく、
“隣に誰かがいる感覚” を演出する。

## 主要機能

- 場面選択（仕事 / 日常）
- お相撲さん同席モード
- 助言メッセージ表示
- 音声再生
- 番付ダッシュボード
- 利用履歴表示

## 日常シーン例

- 飲み会翌朝
- 返信できないLINE
- 親への電話
- 病院予約
- 「特に理由はないけど不安」

## UI 方針

- 常にお相撲さんが画面隅にいる
- 無言待機モードを持つ
- 過度に便利ツール化しない
- 「助言」より「存在感」を優先する

## ディレクトリ

```text
frontend/
```

---

# U-02: orchestrator（親方）

## 説明

全体の進行を管理する中心ユニット。

ユーザー状況を見ながら、
どの技を出すか、
いつ声をかけるかを判断する。

## 主要機能

- セッション開始 / 終了
- シーン判別
- 相撲技選択
- 助言生成トリガー
- WebSocket 通知
- 同席状態管理

## 特徴

単なる API 制御ではなく、
「お相撲さんの間合い」を管理する役割。

## ディレクトリ

```text
backend/orchestrator/
```

---

# U-03: encouragement（決まり手）

## 説明

お相撲さんの “声” を作るユニット。

単なるアドバイス生成ではなく、
「一緒に耐えてくれる感覚」を生み出す。

## 主要機能

- Bedrock による助言生成
- 相撲技に応じた口調切替
- Polly 音声生成
- 音声ファイル保存
- リアルタイム励まし

## 仕事シーン例

- 「押し出しで前へ出ましょう」
- 「まわしは取られておりません」

## 日常シーン例

- 「まず一言で大丈夫です」
- 「まだ土俵に残っています」
- 「今日は四つ身でいきましょう」

## 方針

“正しい助言” より、
「言われると少し動ける言葉」を優先する。

## ディレクトリ

```text
backend/encouragement/
```

---

# U-04: analysis（見合い）

## 説明

ユーザーの “崩れそうな瞬間” を検知するユニット。

## 主要機能

- リアルタイム音声認識
- 緊張ワード検知
- 沈黙検知
- 早口検知
- ため息・間の検知
- 不安ワード検知

## 検知対象

### 仕事

- 「申し訳ありません」
- 「困ります」
- 沈黙
- 早口

### 日常

- 「返したくない」
- 「怒ってるかも」
- 「電話したくない」
- 「今日は無理」

## 方針

“ユーザーが限界に近づいた瞬間” を見つける。

## ディレクトリ

```text
backend/analysis/
```

---

# U-05: dependency-tracker（番付昇進）

## 説明

ユーザーが、
どれだけお相撲さんに頼るようになったかを記録する。

## 主要機能

- 利用履歴保存
- 番付スコア計算
- 昇進判定
- 利用比率分析
- 特別メッセージ生成

## 番付

| 番付 | 状態 |
|---|---|
| 序ノ口 | 「ちょっと安心する」 |
| 三段目 | 「会話前に呼ぶようになる」 |
| 幕下 | 「いないと不安」 |
| 十両 | 「毎日呼ぶ」 |
| 幕内 | 「一人で始められない」 |
| 横綱 | 「もう離れられませんな」 |

## 方針

依存を “悪いもの” として扱わない。

ユーザーが少しずつ、
「一人で向き合えなくなる」過程を、
番付として可視化する。

## ディレクトリ

```text
backend/dependency-tracker/
```
