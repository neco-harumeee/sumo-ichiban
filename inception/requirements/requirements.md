# 要件定義書 — 相撲一番（SUMO ICHIBAN）

## インテント分析サマリー

| 項目 | 内容 |
|------|------|
| **ユーザーリクエスト** | 仕事・日常生活のあらゆる場面に AI のお相撲さんが四つに組み、精神的に支え続けることで依存を生む実用的 AI アシスタント |
| **リクエストタイプ** | 新規プロジェクト（Greenfield） |
| **スコープ** | 複数コンポーネント（Web UI + AI エージェント + 音声認識 + 音声生成） |
| **複雑度** | Moderate（AWS サービス複数連携、ハッカソン期間内に完結） |

---

## サービス概要

**相撲一番** は、仕事のクレーム電話・気の重い会議・言いづらい相談から、夕飯の献立・買い物リスト・体調管理・スケジュール調整まで、日常のあらゆる「ちょっと困った場面」に Amazon Bedrock を中心とした AI のお相撲さんが四つに組むサービス。

問題を解決するのではなく、一緒に土俵に立ってくれる。

利用者は安心するが、次第に「お相撲さん無しでは何も決められない」状態になっていく。

---

## 機能要件

### FR-01: 場面選択 UI
- ユーザーが Web アプリ上で同席してほしい場面を選択できる

  **仕事シーン**
  - クレーム電話 — **押し出し**（相手の圧力を受け止め、前に進む）
  - 1on1での交渉 — **寄り切り**（相手が押されていることに気づかない技）
  - Zoom 会議 — **突き押し**（発言のタイミングを逃さない）
  - 苦手な相談 — **掬い投げ**（相手の懐に入って話を引き出す）
  - 気まずい報告 — **上手投げ**（状況を大きく動かす一手）

  **日常生活シーン**
  - 夕飯の献立決め（引き落とし）
  - 買い物リスト管理（叩き込み）
  - 体調・健康管理（土俵際の踏ん張り）
  - スケジュール調整（割り出し）
  - 人間関係の悩み（抱え込み）

- シンプルなハッカソンデモ用画面（認証なし）
- 選択後、AI お相撲さんが起動する

### FR-02: 仕事サポートモード（押し出し・寄り切り・突き押し）
- 通話・会議中の緊張を検知すると助言メッセージをリアルタイム表示
  - 🍵「落ち着いております。まわしは取られておりません」
  - 🧂「土俵際の踏ん張りどころです」
  - 🍲「ここは寄り切りで押し込みましょう」
- Amazon Transcribe でリアルタイム音声認識・テキスト化し、トリガーワードを検知してサポートを起動する
  - トリガーワード例（クレーム電話）: 「申し訳」「すみません」「困ります」「ちょっと待って」
  - トリガーワード例（1on1での交渉）: 「えーと」「あの」「難しいですね」「ちょっと」
  - トリガーワード例（夫婦の会話）: 「やっぱり」「いいや」「別に」「もういい」「すみません」
  - 沈黙（3秒以上の無音）もトリガーとして機能する
- Amazon Bedrock が状況に応じた助言テキストを生成

### FR-03: 日常生活サポートモード（引き落とし・叩き込み・割り出し）
- テキスト入力または音声入力で日常の悩みを受け付ける
- 以下のシーンに対応:
  - **献立提案**: 冷蔵庫の食材から最適な献立を提案（引き落とし）
  - **買い物リスト**: 必要なものを整理・管理（叩き込み）
  - **体調管理**: 症状から対処法を提案（土俵際の踏ん張り）
  - **スケジュール調整**: 込み入った予定を整理（割り出し）
  - **人間関係**: 悩みを受け止めてアドバイス（抱え込み）
- メッセージ例:
  - 🍲「冷蔵庫の食材で上手投げを決めましょう」
  - 🧂「買い忘れを叩き込みで防ぎます」
  - 🍵「体調の乱れ、土俵際で踏ん張りましょう」

### FR-04: 会議同席モード（突き押し・引き落とし）
- Zoom 等の会議画面の隅に常駐表示
- 以下を検知して助言メッセージを表示
  - 沈黙（一定時間の無音）
  - 発話速度の上昇（Transcribe の文字起こし結果から早口を検知）
  - 長時間ミュート
  - 緊張ワードの頻出（「えーと」「あの」「すみません」等）
- メッセージ例:
  - 🍵「突き押しで発言のタイミングを掴みましょう」
  - 🍲「長丁場ですな。まだ土俵にいます」
  - 🧂（塩を撒くアニメーション）

### FR-05: 依存度トラッキング（番付制度）
- 利用回数・利用時間・依存スコアを記録
- 「お相撲さん依存度」を番付形式でダッシュボードに表示
- 依存度が上がるほど「もう離れられませんな」等のメッセージを表示
- 番付: 序ノ口 → 序二段 → 三段目 → 幕下 → 十両 → 幕内 → 横綱

### FR-06: お相撲さん音声（Polly）
- Amazon Polly でお相撲さんの助言音声（MP3）を生成
- 状況検知時に音声を再生
- 音声は S3 に一時保存

### FR-07: AI 助言・励まし生成
- Amazon Bedrock（Claude 3 系）が状況に応じた助言テキストを生成
- 相撲技・相撲用語を使用した実用的なアドバイス
- 文脈（仕事 / 日常生活 / 健康 / 人間関係）に応じてトーンを調整
- 仕事シーンでは具体的な対処法、日常シーンでは実用的な提案を含める

## 非機能要件

### NFR-01: AWS サービス構成
- **Amazon Bedrock** — 助言・励ましテキスト生成（Claude 3 系）
- **Amazon Transcribe** — リアルタイム音声認識・テキスト化。テキストからトリガーワードを検知してサポートを起動
- **Amazon Polly** — お相撲さん音声合成（MP3 出力）
- **AWS Lambda** — サーバーレス処理基盤
- **Amazon API Gateway** — Web UI からのリクエスト受付（REST + WebSocket）
- **Amazon S3** — 生成した音声ファイルの一時保存（TTL: 1時間）
- **Amazon DynamoDB** — セッション状態・依存度スコア管理

### NFR-02: パフォーマンス・可用性
- **応答レイテンシ**: 状況検知から助言メッセージ表示まで **3秒以内**を目標（突き押しの速さで）
- **Polly 音声生成**: リクエストから再生開始まで **5秒以内**
- **可用性**: ハッカソンデモ期間中の稼働を確保（SLA 定義なし・PoC）
- **同時接続数**: デモ想定 **最大10セッション**（スケールアウト設計は不要）
- **Lambda タイムアウト**: Orchestrator 30秒 / Analysis 60秒（Transcribe ストリーミング）

### NFR-03: セキュリティ・プライバシー
- 認証なし（ハッカソンデモ用 PoC）
- **会話データは保存しない** — Transcribe の文字起こし結果はメモリ内処理のみ、DynamoDB には保存しない
- **音声ファイルの TTL** — S3 に保存する Polly 生成 MP3 は **1時間後に自動削除**（S3 Lifecycle）
- **個人情報の非永続化** — 氏名・連絡先等の個人情報は一切保存しない
- **DynamoDB 保存データ** — セッション ID・場面・時間・サポート回数・依存度スコアのみ（発話内容は含まない）
- セキュリティ拡張ルール: **スキップ**（PoC のため）

### NFR-04: コスト制約
- **Bedrock** — Claude 3 Haiku を優先使用（Sonnet はフォールバック）。1セッションあたりの推定コスト: **$0.01 以下**
- **Transcribe** — ストリーミング利用。デモ期間の総利用時間: **10時間以内**を目安
- **Polly** — 標準音声を使用。1リクエストあたり最大500文字
- **月間コスト上限目安**: **$50以下**（ハッカソンデモ期間）
- **CloudWatch アラーム**: Bedrock / Transcribe の利用コストが閾値を超えた場合に通知

### NFR-05: 運用性・障害時の挙動
- **CloudWatch Logs** — 全 Lambda のログを記録（エラー・レイテンシ）
- **Bedrock 応答失敗時** — プリセットの助言メッセージ（固定文言）にフォールバック
- **Transcribe 接続断時** — フロントエンドに「音声認識を再接続中」を表示し、自動リトライ（最大3回）
- **Lambda タイムアウト時** — エラーレスポンスを返し、フロントエンドは次の検知イベントを待機
- **デプロイ**: AWS CDK による再現可能なデプロイ（`cdk deploy` 1コマンド）

### NFR-06: テスト方針
- プロパティベーステスト: **スキップ**（PoC のため）
- 基本的な動作確認テストのみ実施

### NFR-07: デモ・提供形態
- Web アプリ（ブラウザ動作、PC・スマホ対応）
- デモ形式: スライド + 動画デモ（録画）
- ライブデモは不要

### NFR-08: 開発スコープ
- ハッカソン期間内に完結する実装量
- コア機能（FR-01〜FR-07）を実装、一部モック実装で工数を抑える

## 成功基準

- Web アプリ上で場面選択 → お相撲さん同席 → 状況検知 → 助言メッセージ表示 → 音声再生 の一連のフローがデモ動画で確認できる
- 仕事シーンと日常生活シーンの両方でデモが成立する
- AWS サービス（Bedrock / Lambda / Transcribe / Polly / API Gateway / S3）を実際に使用している
- 依存度スコア（番付）が上昇していく様子がデモで確認できる

## KPI / Success Metrics

### ユーザー価値の指標

| KPI | 目標値 | 計測方法 |
|-----|--------|---------|
| 1セッション平均時間 | **5分以上** | DynamoDB の duration_sec 平均 |
| セッション完走率（途中離脱なし） | **80%以上** | end_session 呼び出し率 |
| リピート利用率（3日以内に再利用） | **50%以上** | DynamoDB の last_session_at 差分 |
| 依存度スコア上昇ユーザー割合 | **70%以上** | dependency_scores テーブルの score 推移 |
| 日常シーン利用率 | **40%以上** | scene カラムの分布 |

## リスク・倫理・ガードレール

### 倫理リスク

| リスク | 対策 |
|--------|------|
| 孤独感への過度な依存 | 依存度「横綱」到達時に「たまには一人で土俵に立ってみましょう」を表示 |
| メンタルヘルス悪化の助長 | センシティブワード（「死にたい」「消えたい」等）を検知した場合は固定応答に切り替え、専門機関への相談を促す |
| 不適切な助言生成 | Bedrock の出力に不適切語句が含まれる場合はフォールバックメッセージに差し替え |
| 医療・法律アドバイスの誤用 | 体調管理シーンでは「医療機関への相談を推奨」を必ず付記。法的判断は行わない |

### プライバシーリスク

| リスク | 対策 |
|--------|------|
| 会話内容の漏洩 | Transcribe の文字起こし結果は DynamoDB に保存しない（メモリ内処理のみ） |
| 音声データの残存 | S3 の Polly 生成 MP3 に TTL 1時間を設定（S3 Lifecycle） |
| 個人情報の蓄積 | 氏名・連絡先等は一切収集・保存しない |

### AI 誤判定リスク

| リスク | 対策 |
|--------|------|
| 緊張ワードの誤検知 | 検知閾値を調整可能にする（デフォルト: 同一ワード2回以上） |
| 沈黙の誤検知（背景音） | 3秒以上の連続無音のみ検知（短い間は無視） |
| Bedrock の不適切生成 | システムプロンプトで相撲用語・実用的助言のみに制約。出力フィルタリングを実装 |
| 日常シーンでの過剰依存促進 | 献立・買い物等の日常シーンでは「自分で決める力」を残す助言スタイルを維持 |

### コスト暴走リスク

| リスク | 対策 |
|--------|------|
| Bedrock 過剰呼び出し | 同一セッション内で30秒以内の連続呼び出しをスロットリング |
| Transcribe ストリーミング放置 | セッション終了時に必ず Transcribe ストリームを停止。Lambda タイムアウトでも強制終了 |
| コスト超過 | CloudWatch アラームで月額 $50 超過時に SNS 通知 |
