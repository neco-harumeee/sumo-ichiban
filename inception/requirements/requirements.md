## インテント分析サマリー

| 項目 | 内容 |

## サービス概要

**お相撲さんと一緒** は、仕事のクレーム電話・気の重い会議・言いづらい相談など「ちょっと嫌な場面」に、Amazon Bedrock を中心とした AI のお相撲さんが同席するサービス。

問題は解決しない。ただ、横で励ましてくれる。

利用者は安心するが、次第に「お相撲さん無しでは電話できない」状態になっていく。

---

## 機能要件

### FR-01: 場面選択 UI
- ユーザーが Web アプリ上で同席してほしい場面を選択できる
  - クレーム電話
  - 上司との面談
  - Zoom 会議
  - 苦手な相談
  - 気まずい報告
- シンプルなハッカソンデモ用画面（認証なし）
- 選択後、AI お相撲さんが起動する

### FR-02: クレーム電話同行モード
- 通話中の緊張を検知すると励ましメッセージをリアルタイム表示
  - 🍵「落ち着いております」
  - 🧂「まわしは取られておりません」
  - 🍲「ここは耐えどころですな」
- Amazon Transcribe でリアルタイム音声認識・緊張ワード検知
- Amazon Bedrock が状況に応じた励ましテキストを生成

### FR-03: 会議同席モード
- Zoom 等の会議画面の隅に常駐表示
- 以下を検知して励ましメッセージを表示
  - 沈黙（一定時間の無音）
  - 発話速度の上昇（Transcribe の文字起こし結果から早口を検知）
  - 長時間ミュート
  - 緊張ワードの頻出（「えーと」「あの」「すみません」等）
- メッセージ例：
  - 🍵「どすこい…」
  - 🍲「長丁場ですな」
  - 🧂（塩を撒くアニメーション）

### FR-04: 依存度トラッキング
- 利用回数・利用時間・依存スコアを記録
- 「お相撲さん依存度」をダッシュボードに表示
- 依存度が上がるほど「もう離れられませんな」等のメッセージを表示

### FR-05: お相撲さん音声
- Amazon Polly でお相撲さんの励まし音声（MP3）を生成
- 緊張検知時に音声を再生
- 音声は S3 に一時保存

### FR-06: AI 励まし生成
- Amazon Bedrock（Claude 3 系）が状況に応じた励ましテキストを生成
- 相撲用語・力士らしい言い回しを使用
- 文脈（クレーム電話 / 会議 / 相談）に応じてトーンを調整

## 非機能要件

### NFR-01: AWS サービス構成
- **Amazon Bedrock** — 励ましテキスト生成（Claude 3 系）
- **Amazon Transcribe** — リアルタイム音声認識・緊張ワード検知・沈黙検知・発話速度解析
- **Amazon Polly** — お相撲さん音声合成（MP3 出力）
- **AWS Lambda** — サーバーレス処理基盤
- **Amazon API Gateway** — Web UI からのリクエスト受付（REST + WebSocket）
- **Amazon S3** — 生成した音声ファイルの一時保存
- **Amazon DynamoDB** — セッション状態・依存度スコア管理

### NFR-02: 認証・セキュリティ
- 認証なし（ハッカソンデモ用 PoC）
- セキュリティ拡張ルール: **スキップ**（PoC のため）

### NFR-03: テスト方針
- プロパティベーステスト: **スキップ**（PoC のため）
- 基本的な動作確認テストのみ実施

### NFR-04: デモ・提供形態
- Web アプリ（ブラウザ動作、PC・スマホ対応）
- デモ形式: スライド + 動画デモ（録画）
- ライブデモは不要

### NFR-05: 開発スコープ
- ハッカソン期間内に完結する実装量
- コア機能（FR-01〜FR-06）を実装、一部モック実装で工数を抑える

## 成功基準

- Web アプリ上で場面選択 → お相撲さん同席 → 緊張検知 → 励ましメッセージ表示 → 音声再生 の一連のフローがデモ動画で確認できる
- AWS サービス（Bedrock / Lambda / Transcribe / Polly / API Gateway / S3）を実際に使用している
- ハッカソン審査員に「笑えないリアリティ」と「技術的完成度」が伝わる
- 依存度スコアが上昇していく様子がデモで確認できる
