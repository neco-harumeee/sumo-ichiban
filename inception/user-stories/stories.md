## エピック 1: トレーニング宣言

### US-01: トレーニング開始宣言
**As a** トレーニング愛好家（田中 剛）  
**I want to** Web アプリ上でトレーニング開始と終了予定時刻を入力したい  
**So that** AI エージェントが自動的に社会的シールドを展開してくれる

**受け入れ基準**:
- [ ] Web 画面に「トレーニング開始」ボタンと終了予定時刻の入力欄が表示される
- [ ] 入力後、「TRAINING LOCK 発動中」の状態表示に切り替わる
- [ ] バックエンド（Lambda）が起動し、Bedrock エージェントが初期化される

---

### US-02: トレーニング終了宣言
**As a** トレーニング愛好家（田中 剛）  
**I want to** トレーニング終了をアプリに伝えたい  
**So that** 社会的シールドが解除され、通常の連絡が再開される

**受け入れ基準**:
- [ ] 「トレーニング終了」ボタンを押すと LOCK が解除される
- [ ] 画面が通常状態に戻る
- [ ] 実行されたアクションのサマリーが表示される（「メール 2 件返信済み」等）

---

## エピック 2: 社会的シールド

### US-03: 仕事メールへの自動返信
**As a** トレーニング愛好家（田中 剛）  
**I want to** トレーニング中に届いた仕事メールを AI が自動返信してほしい  
**So that** 「メール返してないな…」という罪悪感なしにトレーニングに集中できる

**受け入れ基準**:
- [ ] Bedrock がメール内容を解析し、文脈に合った返信文を生成する
- [ ] 「現在外出対応中です。後ほど改めてご連絡します。」等の自然な文面が生成される
- [ ] デモ動画では実際に送信される（コードはモック）

---

### US-04: 仕事メール返信文のカスタマイズ
**As a** トレーニング愛好家（田中 剛）  
**I want to** 返信のトーン（丁寧 / 簡潔）を事前に設定したい  
**So that** 自分のスタイルに合った返信が自動送信される

**受け入れ基準**:
- [ ] Web UI に返信トーン設定（丁寧 / 標準 / 簡潔）が存在する
- [ ] 設定に応じて Bedrock が異なるトーンの文面を生成する

---

### US-05: 家族への自動連絡
**As a** トレーニング愛好家（田中 剛）  
**I want to** 家族に「買い物して帰ります」等の連絡を AI が自動送信してほしい  
**So that** 家族への気遣いを忘れずにトレーニングに集中できる

**受け入れ基準**:
- [ ] 事前登録した家族の連絡先に、適切なタイミングで連絡が送信される
- [ ] Bedrock が時刻と状況に応じた文面を生成する（例: 夕方なら「夕食は外で食べます」）
- [ ] デモ動画では実際に送信される（コードはモック）

---

### US-06: 連絡先・メッセージテンプレートの事前登録
**As a** トレーニング愛好家（田中 剛）  
**I want to** 家族の連絡先と送信タイミングを事前に設定したい  
**So that** トレーニング中に毎回設定しなくてよい

**受け入れ基準**:
- [ ] Web UI に連絡先（メールアドレス）と送信タイミング（開始後 X 分）を登録できる
- [ ] 登録内容がセッション間で保持される（DynamoDB 等）

## エピック 3: モチベ反論機能

### US-07: サボり検知とモチベ反論の生成
**As a** トレーニング愛好家（田中 剛）  
**I want to** 宣言した終了時刻を過ぎてもトレーニングしていない場合に AI から反論を受けたい  
**So that** 言い訳を論破されてトレーニングを再開できる

**受け入れ基準**:
- [ ] 宣言終了時刻を X 分過ぎた場合、サボり検知がトリガーされる
- [ ] Bedrock が「雨の日は空気抵抗が減ります」等の圧強めの反論テキストを生成する
- [ ] 生成されたテキストが画面に表示される

---

### US-08: モチベ反論の音声送信
**As a** トレーニング愛好家（田中 剛）  
**I want to** 反論テキストを音声ファイルとしてメール / LINE で受け取りたい  
**So that** 逃げ場なく反論を聞かされる

**受け入れ基準**:
- [ ] Amazon Polly が反論テキストを MP3 に変換する
- [ ] MP3 ファイルが S3 に保存される
- [ ] メール / LINE に音声ファイルが添付されて送信される（デモ動画で確認）

---

### US-09: 反論の圧の強さ設定
**As a** トレーニング愛好家（田中 剛）  
**I want to** 反論の「圧の強さ」を設定したい  
**So that** 自分に合ったモチベーション強度で反論を受けられる

**受け入れ基準**:
- [ ] Web UI に圧の強さ設定（弱め / 標準 / 圧強め）が存在する
- [ ] 設定に応じて Bedrock が異なるトーンの反論を生成する（圧強めは命令口調）

---

## エピック 4: Amazon 物流連携

### US-10: トレーニングギアの自動発注
**As a** トレーニング愛好家（田中 剛）  
**I want to** AI がトレーニングギアを自動的に Amazon で発注してほしい  
**So that** 「ギアが古い」という言い訳を事前に潰してもらえる

**受け入れ基準**:
- [ ] Bedrock が発注対象のギア（シューズ等）を判断する
- [ ] PA-API 経由で商品検索・発注が実行される（デモ動画で確認、コードはモック）
- [ ] 発注後「シューズ、ぽちっとしておきました。」の通知がユーザーに届く

---

### US-11: 発注履歴の確認
**As a** トレーニング愛好家（田中 剛）  
**I want to** AI が発注したものの一覧を確認したい  
**So that** 何が注文されたか把握できる

**受け入れ基準**:
- [ ] Web UI に発注履歴（商品名・日時・金額）が表示される
- [ ] 各セッションで発注されたアイテムが記録されている
